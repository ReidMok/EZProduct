# 📋 需要查看的信息

## 🔍 在 Network 标签页中查找

### 步骤1：过滤 localhost:3000 的请求

1. 在 Network 标签页的**搜索框**中输入：`localhost:3000`
2. 或者点击过滤器，选择 "Fetch/XHR" 或 "Doc"
3. 查看是否有对 `localhost:3000` 的请求

### 步骤2：如果找到了请求

点击该请求，然后查看：

#### A. Headers 标签页
- **Request URL**：完整的 URL 是什么？（应该包含 `?embedded=1&session=...&shop=...` 等参数）
- **Status Code**：状态码是什么？
  - `200` = 成功
  - `404` = 未找到
  - `500` = 服务器错误
  - `(failed)` = 请求失败
- **Response Headers**：查看以下响应头：
  - `Content-Type`：应该是什么？
  - `Content-Security-Policy`：是否有这个头？
  - `X-Frame-Options`：是否有这个头？

#### B. Response 标签页
- **响应内容是什么？**
  - 如果是 HTML，显示的是什么？
  - 如果是错误信息，请复制给我
  - 如果是空的，说明服务器没有返回内容

#### C. Preview 标签页
- 如果有内容，显示的是什么？

### 步骤3：如果没有找到请求

如果搜索 `localhost:3000` 后没有找到任何请求，说明：
- 请求可能被浏览器安全策略阻止了
- 请求可能根本没有发出
- 可能需要查看 Console 标签页的错误信息
